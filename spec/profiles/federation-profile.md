# ODTIS Federation profile

**Profile ID:** `federation`

**Depends on:** `trust-network`, `core-identity`

**ODTIS sections:** 6

---

## Purpose

Bilateral federation between **independent trust network instances** with **non-transitive** trust. Governs institutional gateway exchange across operators - not browser SSO between IdPs.

**Depends on:** Trust Network + Core Identity.

**Normative scope:** section 6 - **6 registry IDs** (`ODTIS-0401`-`0406`).

## Deployment phase applicability

| Phase | Federation expectation |
|-------|------------------------|
| 1-2 | MUST NOT claim Federation profile (single-instance topology) |
| 3 | Agreement design + sandbox bilateral tests |
| 4 | Production federated routes with L3 operator targets |

Conformance statement MUST list active bilateral agreements when Federation is claimed (`ODTIS-0403`).

---

## Not OpenID Federation

This profile is **not** [OpenID Federation 1.1](https://openid.net/developers/specs/). See [OIDF positioning](/governance/liaison/OIDF-POSITIONING/).
